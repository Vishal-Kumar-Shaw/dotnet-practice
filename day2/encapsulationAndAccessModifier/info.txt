Q. What is encapsulation?
Ans. Binding data and methods into a class is called encapsulation. Data is make private so that it cannot be changed directly by any object
and for controlling the data i.e. getting and setting, we create getter and setter methods. This is called encapsulation
Encapsulation = control data using methods, not direct access.

Q. Why we use private field?
Ans. For security purpose, we use private field so that data can't be changed directly
To protect data from invalid or unauthorized changes.

Q. Internal vs Protected
Ans. Internal - same assembly
Protected - derived classes
