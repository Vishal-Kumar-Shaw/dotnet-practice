✅ 1️⃣ Abstract class me constructor kyun hota hai?
✔ Tumhara logic:

Bilkul sahi — abstract class ka constructor child ke initialization ke liye hota hai.

⭐ PERFECT INTERVIEW ANSWER:

Abstract class me constructor isliye hota hai kyunki
jab child class inherit karti hai, tab parent ke fields/properties ko initialize karna zaroori hota hai.

Abstract class ka object nahi banta,
par child object banne par parent constructor pehle call hota hai:

public Dog() : base("Animal") { }


Iska purpose:
✔ common initialization
✔ base fields set karna
✔ code duplication avoid karna


✅ 2️⃣ Interface me fields kyun allowed nahi?
✔ Tumhara logic: good

Interface = only rules, no implementation.

⭐ PERFECT INTERVIEW ANSWER:

Interface ek pure contract hota hai.
Uska kaam sirf yeh batana hai ki:

"Class ko kaunse methods implement karne hain."

Interface:

❌ object nahi bana sakta
❌ koi state store nahi kar sakta
❌ koi field hold nahi kar sakta

Isliye:

No instance fields

No constructors

No memory for data

Interface = 100% behavior, 0% data



✅ 3️⃣ Interface multiple inheritance kyu allow karta hai?
✔ Tumhara idea: good

Rules multiple jagah se aa sakte hain.

⭐ PERFECT INTERVIEW ANSWER:

Classes me multiple inheritance dangerous hota hai
(diamond problem, ambiguity etc.)

But interface me koi implementation nahi hoti,
sirf method signatures hote hain.

Isliye:

✔ No diamond problem
✔ No conflict
✔ No ambiguity

Example:
A class can be both:

IAnimal AND IPet


Kyuki dono me sirf rules hain.

Interface multiple inheritance = safe + flexible.
